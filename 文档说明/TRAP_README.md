# 陷入指令模拟程序

## 项目简介

本项目实现了完整的操作系统陷入（Trap）处理机制，包括异常、系统调用和硬件中断的模拟。展示了从陷入触发到处理完成的完整流程。

**适用场景：**
- 操作系统课程学习
- 理解陷入处理机制
- 理解中断向量表
- 理解陷入优先级和嵌套
- 教学演示和实验

## 核心功能

### 1. 陷入类型分类

**异常（Exception）- 同步，程序执行引起**
- ✓ 除零异常（DIVIDE_BY_ZERO）
- ✓ 溢出异常（OVERFLOW）
- ✓ 非法指令（INVALID_OPCODE）
- ✓ 一般保护异常（GENERAL_PROTECTION）
- ✓ 缺页异常（PAGE_FAULT）
- ✓ 栈溢出（STACK_OVERFLOW）

**系统调用（Trap）- 同步，程序主动触发**
- ✓ 系统调用陷入（SYSCALL - int 0x80）

**硬件中断（Interrupt）- 异步，外部设备触发**
- ✓ 时钟中断（TIMER）
- ✓ 键盘中断（KEYBOARD）
- ✓ 磁盘中断（DISK）
- ✓ 网络中断（NETWORK）

### 2. 陷入向量表（IVT）

- ✓ 陷入处理程序注册
- ✓ 陷入优先级管理
- ✓ 陷入屏蔽机制
- ✓ 处理程序查找

### 3. 陷入处理流程

```
触发陷入 → 保存现场 → 切换内核态 → 查陷入向量表 
→ 执行处理程序 → 恢复现场 → 切换用户态 → 返回
```

### 4. 高级特性

- ✓ 陷入嵌套处理
- ✓ 中断使能/禁止（CLI/STI）
- ✓ 陷入优先级（CRITICAL/HIGH/MEDIUM/LOW）
- ✓ 陷入统计
- ✓ 上下文保存/恢复

## 陷入类型详解

### 异常（Exception）

| 陷入号 | 名称 | 优先级 | 说明 |
|-------|------|--------|------|
| 0x00 | DIVIDE_BY_ZERO | HIGH | 除零异常 |
| 0x01 | OVERFLOW | HIGH | 溢出异常 |
| 0x02 | INVALID_OPCODE | HIGH | 非法指令 |
| 0x03 | GENERAL_PROTECTION | HIGH | 一般保护异常 |
| 0x0C | STACK_OVERFLOW | HIGH | 栈溢出 |
| 0x0E | PAGE_FAULT | HIGH | 缺页异常 |

### 系统调用（Trap）

| 陷入号 | 名称 | 优先级 | 说明 |
|-------|------|--------|------|
| 0x80 | SYSCALL | LOW | 系统调用（int 0x80） |

### 硬件中断（Interrupt）

| 陷入号 | 名称 | 优先级 | 说明 |
|-------|------|--------|------|
| 0x20 | TIMER | MEDIUM | 时钟中断 |
| 0x21 | KEYBOARD | MEDIUM | 键盘中断 |
| 0x2E | DISK | MEDIUM | 磁盘中断 |
| 0x2F | NETWORK | MEDIUM | 网络中断 |

## 陷入处理流程图

```
┌─────────────────────────────────────────────────────────┐
│                    用户程序执行                          │
│  执行指令 / 触发异常 / 外部中断                          │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 陷入触发
┌─────────────────────────────────────────────────────────┐
│                   检查陷入条件                            │
│  - 是否被屏蔽？                                          │
│  - 中断是否使能？                                        │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 保存现场
┌─────────────────────────────────────────────────────────┐
│                   保存CPU上下文                           │
│  - 保存所有寄存器（EAX, EBX, EIP, ESP...）              │
│  - 创建陷入帧（TrapFrame）                               │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 切换特权级
┌─────────────────────────────────────────────────────────┐
│                   用户态 → 内核态                         │
│  - 切换CPU模式                                           │
│  - 压入陷入帧到陷入栈                                     │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 查陷入向量表
┌─────────────────────────────────────────────────────────┐
│                 陷入向量表（IVT）                         │
│  根据陷入号查找处理程序                                   │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 执行处理程序
┌─────────────────────────────────────────────────────────┐
│                 陷入处理程序                              │
│  - 异常处理：终止进程/发送信号                            │
│  - 系统调用：执行内核服务                                 │
│  - 硬件中断：处理I/O完成/调度                            │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 恢复现场
┌─────────────────────────────────────────────────────────┐
│                   恢复CPU上下文                           │
│  - 弹出陷入帧                                            │
│  - 恢复寄存器状态                                        │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 切换特权级
┌─────────────────────────────────────────────────────────┐
│                   内核态 → 用户态                         │
│  （如果陷入栈为空）                                       │
└────────────────────┬────────────────────────────────────┘
                     │
                     ↓ 返回
┌─────────────────────────────────────────────────────────┐
│                    继续执行                              │
│  返回到被中断的位置继续执行                               │
└─────────────────────────────────────────────────────────┘
```

## 核心组件详解

### 1. CPU类

模拟CPU的状态和陷入处理能力。

**关键属性：**
- `mode` - CPU模式（用户态/内核态）
- `context` - CPU上下文（寄存器）
- `trap_stack` - 陷入栈（支持嵌套）
- `interrupt_enabled` - 中断使能标志

**关键方法：**
- `switch_to_kernel_mode()` - 切换到内核态
- `switch_to_user_mode()` - 切换回用户态
- `disable_interrupt()` - 关中断（CLI）
- `enable_interrupt()` - 开中断（STI）
- `push_trap_frame()` - 压入陷入帧
- `pop_trap_frame()` - 弹出陷入帧

### 2. TrapVectorTable类

陷入向量表，管理所有陷入处理程序。

**关键方法：**
- `register()` - 注册陷入处理程序
- `get_handler()` - 获取处理程序
- `get_priority()` - 获取优先级
- `mask_trap()` - 屏蔽陷入
- `unmask_trap()` - 取消屏蔽
- `is_masked()` - 检查是否被屏蔽

### 3. TrapHandler类

陷入处理器，核心陷入处理逻辑。

**关键方法：**
- `trigger_trap()` - 触发陷入
- `handle_divide_by_zero()` - 处理除零异常
- `handle_page_fault()` - 处理缺页异常
- `handle_syscall()` - 处理系统调用
- `handle_timer_interrupt()` - 处理时钟中断
- `handle_keyboard_interrupt()` - 处理键盘中断

### 4. TrapFrame类

陷入帧，保存陷入发生时的完整状态。

**关键属性：**
- `trap_type` - 陷入类型
- `trap_number` - 陷入号
- `error_code` - 错误码
- `context` - CPU上下文
- `timestamp` - 时间戳

## 使用示例

### 示例1：触发异常

```python
from trap_instruction_simulation import CPU, TrapHandler, TrapType

# 初始化
cpu = CPU()
handler = TrapHandler(cpu)

# 触发除零异常
handler.trigger_trap(TrapType.DIVIDE_BY_ZERO)

# 触发缺页异常
handler.trigger_trap(TrapType.PAGE_FAULT, error_code=0x12345000)
```

### 示例2：系统调用

```python
# 设置系统调用参数
cpu.context.eax = 1  # write系统调用
cpu.context.ebx = 1  # stdout
cpu.context.ecx = 0x1000  # buffer地址

# 触发系统调用
handler.trigger_trap(TrapType.SYSCALL)
```

### 示例3：硬件中断

```python
# 时钟中断
handler.trigger_trap(TrapType.TIMER)

# 键盘中断
handler.trigger_trap(TrapType.KEYBOARD)
```

### 示例4：中断屏蔽

```python
# 屏蔽键盘中断
handler.ivt.mask_trap(TrapType.KEYBOARD.value)

# 触发被屏蔽的中断（不会执行）
handler.trigger_trap(TrapType.KEYBOARD)

# 取消屏蔽
handler.ivt.unmask_trap(TrapType.KEYBOARD.value)
```

### 示例5：关中断

```python
# 关中断
cpu.disable_interrupt()

# 触发硬件中断（不会执行）
handler.trigger_trap(TrapType.TIMER)

# 开中断
cpu.enable_interrupt()
```

## 关键知识点

### 1. 陷入的分类

**同步陷入（Synchronous）：**
- 由程序执行引起
- 可预测、可重现
- 例如：异常、系统调用

**异步陷入（Asynchronous）：**
- 由外部事件引起
- 不可预测
- 例如：硬件中断

### 2. 陷入优先级

```
CRITICAL (最高) - 硬件故障
    ↓
HIGH - 异常（除零、缺页、非法指令）
    ↓
MEDIUM - 硬件中断（时钟、键盘、磁盘）
    ↓
LOW (最低) - 系统调用
```

### 3. 陷入处理步骤

1. **保存现场** - 保存CPU上下文到陷入帧
2. **切换特权级** - 用户态 → 内核态
3. **查向量表** - 根据陷入号找处理程序
4. **执行处理** - 调用陷入处理程序
5. **恢复现场** - 从陷入帧恢复上下文
6. **切换特权级** - 内核态 → 用户态
7. **返回** - 继续执行

### 4. 陷入嵌套

- 在处理一个陷入时，可能发生另一个陷入
- 使用陷入栈管理嵌套层次
- 高优先级陷入可以打断低优先级陷入
- 返回时按照后进先出（LIFO）顺序

### 5. 中断屏蔽

**两种屏蔽方式：**

1. **全局屏蔽（CLI/STI）：**
   - 关闭所有硬件中断
   - 异常和系统调用不受影响

2. **选择性屏蔽：**
   - 屏蔽特定陷入号
   - 其他陷入正常处理

## 常见问题

### Q1: 陷入和中断有什么区别？

A: 
- **陷入（Trap）**：广义概念，包括异常、系统调用和硬件中断
- **中断（Interrupt）**：狭义概念，特指硬件中断
- 本程序中，陷入是总称，包含所有类型

### Q2: 为什么异常优先级高于硬件中断？

A: 异常通常表示严重错误（如除零、缺页），必须立即处理，否则程序无法继续执行。硬件中断可以稍后处理。

### Q3: 陷入嵌套的最大层数是多少？

A: 理论上无限制，但实际受限于陷入栈大小。本程序使用Python列表，无实际限制。

### Q4: 关中断后异常还能触发吗？

A: 可以。关中断（CLI）只影响硬件中断，异常和系统调用不受影响。

### Q5: 如何验证真实的陷入处理？

A: 在Linux上可以使用：
```bash
# 查看中断统计
cat /proc/interrupts

# 追踪系统调用
strace -e trace=all your_program

# 触发异常
python3 -c "1/0"  # 除零异常
```
