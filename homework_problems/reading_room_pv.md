# 阅览室问题 - PV操作实现

## 问题描述

某大学图书馆阅览室共有100个座位，读者进入时必须先在入口处登记，读者离开时必须在出口处注销登记。当阅览室满员时，后来的读者需要在入口处等待，当有读者离开后，等待的读者才能进入。

## 问题分析

### 1. 识别共享资源
- **共享资源**：100个座位

### 2. 识别同步关系
- **资源约束**：座位数量有限（100个）
- **同步关系**：
  - 进入需要有空座位
  - 离开释放座位

### 3. 确定信号量

| 信号量 | 类型 | 初值 | 用途 |
|--------|------|------|------|
| `seats` | 资源信号量 | 100 | 表示可用座位数量 |

**说明**：
- 只需要一个资源信号量即可
- 不需要互斥信号量，因为P/V操作本身是原子的
- 登记和注销操作可以通过P/V操作隐式完成

## 伪代码实现

### 信号量定义

```
semaphore seats = 100;  // 可用座位数，初值为100
```

### 读者进程

```
Reader() {
    while (true) {
        // 在外面活动
        do_other_things();
        
        // === 进入阅览室 ===
        P(seats);              // 申请一个座位（登记）
                               // 如果seats=0（满员），则阻塞等待
        
        // 进入阅览室
        enter_reading_room();
        
        // 在阅览室内阅读
        read_books();
        
        // === 离开阅览室 ===
        leave_reading_room();
        
        V(seats);              // 释放一个座位（注销）
                               // 唤醒等待的读者
    }
}
```

## 流程分析

### 进入流程
```
1. 读者到达入口
   ↓
2. P(seats) - 申请座位
   ├─ 如果 seats > 0：获得座位，seats--，进入
   └─ 如果 seats = 0：阻塞等待
   ↓
3. 登记并进入阅览室
   ↓
4. 阅读
```

### 离开流程
```
1. 读者准备离开
   ↓
2. 离开阅览室
   ↓
3. V(seats) - 释放座位
   ├─ seats++
   └─ 如果有读者等待：唤醒一个等待的读者
   ↓
4. 注销完成
```

## 正确性验证

### 1. 互斥性
- ✓ 不需要互斥（每个读者独立占用座位）

### 2. 同步性
- ✓ 满员时读者阻塞（P(seats)在seats=0时阻塞）
- ✓ 有人离开后等待者能进入（V(seats)唤醒等待者）

### 3. 资源约束
- ✓ 阅览室内读者数量 ≤ 100
- ✓ 证明：seats初值=100，每次P操作-1，每次V操作+1
  - 在任意时刻：seats + 室内人数 = 100
  - 因此：室内人数 = 100 - seats ≤ 100

### 4. 无死锁
- ✓ 只有一个信号量，不会形成循环等待

### 5. 无饥饿
- ✓ 等待队列按FIFO顺序唤醒（取决于信号量实现）

## 关键点总结

1. **信号量选择**：
   - 只需一个资源信号量
   - 初值 = 资源总数（100）

2. **操作对应**：
   - P(seats) = 登记 + 申请座位
   - V(seats) = 注销 + 释放座位

3. **简洁性**：
   - 这是最简单的同步问题之一
   - 标准的资源计数模式

## 扩展思考

### 如果需要统计当前人数？

```
semaphore seats = 100;
semaphore mutex = 1;    // 保护计数器
int count = 0;          // 当前人数

Reader() {
    P(seats);           // 申请座位
    
    P(mutex);           // 保护计数器
    count++;
    print("当前人数:", count);
    V(mutex);
    
    read_books();
    
    P(mutex);
    count--;
    print("当前人数:", count);
    V(mutex);
    
    V(seats);           // 释放座位
}
```

### 如果要区分VIP读者？

```
semaphore seats = 100;
semaphore vip_queue = 1;    // VIP优先队列
semaphore normal_queue = 1; // 普通队列

VIP_Reader() {
    P(vip_queue);
    P(seats);
    V(vip_queue);
    
    read_books();
    
    V(seats);
}

Normal_Reader() {
    P(normal_queue);
    P(vip_queue);       // 确保VIP优先
    V(vip_queue);
    P(seats);
    V(normal_queue);
    
    read_books();
    
    V(seats);
}
```

## 总结

阅览室问题是一个**标准的资源计数问题**：
- 模式：有限资源的申请和释放
- 信号量：1个资源信号量
- 初值：资源总数
- 操作：P申请，V释放
- 应用：停车场、连接池、线程池等
